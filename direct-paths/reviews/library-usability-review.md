# Skills Library Review Template

Use this template to assess the library's usability, quality, and completeness.

---

## Quick Assessment

| Category | Score (0-25) | Notes |
|----------|--------------|-------|
| **Usability** | /25 | Can I find what I need? |
| **Sensibility** | /25 | Is content actionable and current? |
| **Flexibility** | /25 | Can skills combine? Is it modular? |
| **Completeness** | /25 | Major gaps? Outdated content? |
| **TOTAL** | /100 | |

---

## 1. Usability Assessment (25 points)

### Navigation Clarity
- [ ] Can find entry point in under 10 seconds
- [ ] Quick Start table covers main use cases
- [ ] "I want to X" → clear path exists
- [ ] Related Skills sections link logically

### Searchability
- [ ] File names are intuitive
- [ ] TL;DR sections provide quick answers
- [ ] Keywords appear where expected

### Questions to Ask
```
"If I want to [X], where do I start?"
"How quickly can I find [Y]?"
"Is the navigation intuitive or confusing?"
```

### Scoring
- 23-25: Excellent (find anything in seconds)
- 18-22: Good (minor friction)
- 12-17: Needs work (confusing paths)
- 0-11: Poor (can't find things)

---

## 2. Sensibility Assessment (25 points)

### Actionable Content
- [ ] Instructions lead to specific outcomes
- [ ] Code examples are copy-paste ready
- [ ] Checklists have clear completion criteria
- [ ] Recommendations are specific, not vague

### Current Information
- [ ] Tools mentioned are available (not deprecated)
- [ ] Dates/versions are 2024-2025 range
- [ ] Pricing/access info is accurate
- [ ] Best practices reflect current industry

### Real Use Cases
- [ ] Examples match actual workflows
- [ ] Skills address real problems
- [ ] Content is practical, not theoretical

### Questions to Ask
```
"Can I follow this and get results?"
"Is this advice current?"
"Does this solve a real problem I have?"
```

### Scoring
- 23-25: Excellent (all actionable, current)
- 18-22: Good (mostly actionable)
- 12-17: Needs work (some outdated/vague)
- 0-11: Poor (theoretical, outdated)

---

## 3. Flexibility Assessment (25 points)

### Modularity
- [ ] Skills work independently
- [ ] Skills combine cleanly
- [ ] No rigid "you must do X" requirements
- [ ] Supports different workflows/preferences

### Adaptability
- [ ] Works for different project types
- [ ] Scales from simple to complex
- [ ] Accommodates tool preferences
- [ ] Malleable, not prescriptive

### Integration
- [ ] Cross-references are helpful
- [ ] Skill combos documented
- [ ] Clear "if X, also consider Y" guidance

### Questions to Ask
```
"Can I use just part of this?"
"Does this force one way of doing things?"
"Can I adapt this to my workflow?"
```

### Scoring
- 23-25: Excellent (fully modular, flexible)
- 18-22: Good (mostly flexible)
- 12-17: Needs work (some rigidity)
- 0-11: Poor (prescriptive, inflexible)

---

## 4. Completeness Assessment (25 points)

### Coverage
- [ ] Major tools/platforms covered
- [ ] Common workflows documented
- [ ] Both technical and creative skills
- [ ] AI tools represented

### Gaps Analysis
```
What's missing?
- [ ] List identified gaps here

What's outdated?
- [ ] List outdated content here

What's duplicated?
- [ ] List redundant content here
```

### Quality Checks
- [ ] TL;DR present in all skills
- [ ] Code examples included where relevant
- [ ] Checklists are actionable
- [ ] Related Skills linked
- [ ] Frontmatter complete (name, description, date)

### Questions to Ask
```
"What major tool is NOT covered?"
"What workflow is missing?"
"What's outdated by more than a year?"
```

### Scoring
- 23-25: Excellent (comprehensive, current)
- 18-22: Good (minor gaps)
- 12-17: Needs work (notable gaps)
- 0-11: Poor (major gaps)

---

## Final Scoring

| Score | Grade | Meaning |
|-------|-------|---------|
| 90-100 | A | Excellent — Ready for production use |
| 80-89 | B | Good — Minor improvements needed |
| 70-79 | C | Acceptable — Notable improvements needed |
| 60-69 | D | Needs work — Significant gaps |
| 0-59 | F | Not ready — Major overhaul needed |

## Quick Review Checklist

For a fast assessment:

```
[ ] Navigation works (can find things quickly)
[ ] TL;DRs are present and useful
[ ] Code examples are copy-paste ready
[ ] Information is current (2024-2025)
[ ] Major tools are covered
[ ] Skills link to related skills
[ ] Checklists are actionable
[ ] No obvious gaps for common workflows
```

If all checked: **Ready to use**
If 1-2 unchecked: **Minor improvements**
If 3+ unchecked: **Needs attention**
