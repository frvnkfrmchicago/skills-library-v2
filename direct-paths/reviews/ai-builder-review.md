# AI Builder Wing - Review Prompt

**Copy this entire prompt to another AI agent for review.**

---

## Context

I have a Skills Library that I use to enhance my AI-assisted development workflow. I recently built an "AI Builder" wing specifically for building AI systems (not just using AI tools to build apps).

The wing has 12 skills:
- python, langchain, agentic-workflows, docker
- golang, huggingface, pytorch, kubernetes
- event-driven, ml-ops, data-pipelines, responsible-ai

## Your Task

Give me a **brutal, honest review** of this AI Builder wing. I need real feedback, not validation.

## What I Care About

### Style & Tone
- I'm a **vibe coder** — I ship fast, iterate, minimal overhead
- These skills should help me move faster, not slow me down with theory
- Patterns should be **copy-paste ready**, not academic explanations
- Should work for solo dev projects AND scale to enterprise when needed

### Technical Quality
- Are the code examples production-ready or toy examples?
- Are versions up-to-date (Python 3.12, PyTorch 2.5, uv, etc.)?
- Do the patterns reflect 2025-2026 best practices?
- Are there anti-patterns or outdated approaches?

### Coverage
- Does this cover what an AI Product Engineer needs?
- What critical skills are missing?
- What's included that shouldn't be?
- Are the cross-references between skills useful?

### Practical Value
- Could someone actually use these skills to build real AI products?
- Are there gaps between reading the skill and actually implementing?
- Do the checklists at the end actually help?

## Evaluation Criteria

Rate each skill 1-5 on:
1. **Actionability** — Can I use this immediately?
2. **Depth** — Is it deep enough without being bloated?
3. **Accuracy** — Are code examples correct and current?
4. **Style fit** — Does it match vibe coder methodology?

## What I Need From You

1. **Overall Grade** (A-F) with justification
2. **Top 3 Strengths** 
3. **Top 3 Weaknesses**
4. **Skills that need work** (with specific suggestions)
5. **Missing skills** that should be added
6. **Skills to remove or merge** (if any)
7. **One thing that would make this 10x better**

## Don't Hold Back

I don't want "this is great!" — I want "this sucks because X, fix it by doing Y."

Be specific. Give examples. Tell me what's wrong.
