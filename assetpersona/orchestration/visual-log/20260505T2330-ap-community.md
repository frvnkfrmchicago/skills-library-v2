## Explanation

This wave closes every named pain point from the user's request. The decisions worth restating:

1. **Comments default to moderated.** Pending → admin or moderator approves → public. This protects Frank from the Discord/Skool problem of waking up to spam in the feed.
2. **No DMs, by design.** The DirectMessage stub is gone (not just disabled). Frank's contact surface is the FAQ "Ask Frank" form, which routes to the existing Inquiries Kanban he already triages every day.
3. **ModeratorGuard exists and is wired.** Promoting a member to moderator now actually gives them moderation access — `/admin/moderation` opens for both admins and mods.
4. **Avatars and covers use Supabase Storage.** Per-user folders with `auth.uid()::text = (storage.foldername(name))[1]` policies — no service-role on the client.
5. **GIFs come from Tenor**, not embedded as iframes. We lift the Tenor URL and store it in our `media_urls`, so deletion stays in our control.
6. **FAQ is a real table** with categories and FTS, not hardcoded blog data. Existing blog `faqSchema` still works and is now rendered visibly on each post.
7. **Account Settings password change is real.** It uses `supabase.auth.updateUser({ password })`. Email change uses the same API and triggers Supabase's two-inbox verification.

Per the Orchestration Librarian Operational Rule, no `tsc --noEmit` or `bun run build` runs were attempted during this multi-agent execution. The user verifies once after the wave settles.
