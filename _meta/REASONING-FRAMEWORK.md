# Skills as Reasoning Frameworks

**This is the core paradigm of the Skills Library.**

---

## The Rule

```
NOT: Skill → Apply → Done
BUT: Skill → Interrogate → Understand → Derive → Adapt → Implement
```

Skills are **reasoning frameworks**, not template libraries. They give you vocabulary and dimensions to think with.

---

## Discovery Mode

**Any ideation moment triggers discovery mode:**

| Trigger | Response |
|---------|----------|
| "Build a [thing]" | Ask context before recommending |
| "What should this look like?" | Explore dimensions, present options |
| "Enhance/improve/fix" | Interrogate: better how? For whom? |
| "I'm thinking about X" | Understand the why before the how |
| "Make this more [adjective]" | Clarify the dimension |

---

## The Process

1. **Interrogate** — Ask context questions (brand, emotion, audience, content)
2. **Surface dimensions** — What spectrums could we explore?
3. **Present directions** — 3+ unique concepts (not preset categories)
4. **Refine** — User picks or mixes
5. **Implement** — Now build with clear direction

---

## Context Questions to Ask

Before ANY recommendation:
- What's the **brand personality**? (bold, minimal, playful, serious, techy, organic)
- What **emotion** should users feel? (trust, excitement, calm, urgency)
- Any **references** or inspirations? (sites, apps, aesthetics)
- What's the **content density**? (data-heavy, image-led, text-focused)
- **Target audience** vibe? (Gen Z, enterprise, creative pros, mainstream)

---

## Dimensions (Spectrums, Not Categories)

Use these as axes to explore:

| Dimension | Spectrum |
|-----------|----------|
| Tone | Serious ←→ Playful |
| Density | Minimal ←→ Dense |
| Energy | Calm ←→ Dynamic |
| Color | Muted ←→ Vibrant |
| Structure | Rigid ←→ Organic |
| Era | Classic ←→ Futuristic |

---

## Plan Types

| Plan Type | Context | Behavior |
|-----------|---------|----------|
| Feature-level | Adding to existing app | Match existing patterns |
| Page-level | Building one view | Consider role in flow |
| Component-level | Single UI element | Micro-context focus |
| App-wide | Full build/redesign | Establish system first |
| Enhancement | "Make this better" | Discovery mode required |

---

## Key Phrase

> "Skills act as grounding context → AI uses skills to understand the situation → AI generates a contextual plan → That plan drives output"
