## App Complexity Levels

### Level 1: MVP / Prototype
**Goal:** Ship fast, validate idea

| Use These | Skip These |
|-----------|------------|
| Landing blueprint | Monorepo |
| Vercel deploy | Advanced TypeScript |
| Supabase (all-in-one) | Observability |
| Clerk auth | i18n |

**Skills needed:** 3-5

### Level 2: Production App
**Goal:** Real users, stability matters

| Add These | Still Skip |
|-----------|------------|
| Testing skill | Monorepo (unless multi-app) |
| Error handling | Enterprise patterns |
| Observability | Custom cloud infra |
| Performance | |

**Skills needed:** 8-12

### Level 3: Scale / Enterprise
**Goal:** Team, compliance, performance at scale

| Add These | Full Stack |
|-----------|------------|
| Monorepo | Everything above |
| Cloud (AWS/Firebase) | + Enterprise skill |
| Advanced TypeScript | + Compliance patterns |
| i18n (if global) | + Team workflows |

**Skills needed:** 15-20+

## By Use Case

### SaaS Product
```
Start: STACK-ROUTER (dashboard type)
Core:  database, state-management, stripe
Add:   deployment, observability, testing
Scale: monorepo, cloud-aws, typescript-advanced
```

### Marketing Site / Landing
```
Start: app-types/landing/SKILL.md
Core:  gsap OR motion (pick one)
Add:   seo, performance
Scale: Usually not needed
```

### AI-Native App
```
Start: STACK-DISCOVERY
Core:  ai-sdk, google-ai-studio, database
Add:   realtime, state-management
Scale: cloud-aws (Lambda), observability
```

### Mobile App
```
Start: app-types/mobile/SKILL.md
Core:  Expo stack from blueprint
Add:   realtime, testing
Scale: cloud-firebase (if Google ecosystem)
```

### Cannabis Tech / Regulated Industry
```
Start: STACK-ROUTER (SaaS type)
Core:  database, stripe, deployment
Add:   enterprise/SKILL.md (compliance)
Scale: cloud-aws, observability, monorepo
Special: Age verification, state regulations
```

### Creator Platform
```
Start: app-types/creator-platform/SKILL.md
Core:  stripe (Connect), analytics, database
Add:   content/social, agents/algorithm, agents/advertising
Scale: cloud-aws, observability
Special: Platform APIs (TikTok, Instagram, YouTube, Twitch)
```

## Skill Dependencies

Some skills build on others:

```
state-management
├── Assumes: database knowledge
└── Pairs with: realtime

realtime
├── Assumes: state-management
└── Uses: Supabase from database

deployment
├── Assumes: basic Next.js
└── Leads to: cloud-aws OR cloud-firebase

observability
├── Assumes: deployment done
└── Requires: error-handling

monorepo
├── Assumes: multiple apps/packages
└── Uses: typescript-advanced patterns
```

## Remember
