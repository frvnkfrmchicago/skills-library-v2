# Client Work

Land clients, scope projects, and deliver without drama.

> **See also:** `workflows/vc-pitch/SKILL.md` for pitching, `agents/technical-leadership/SKILL.md` for leadership

---

## Context Questions

Before taking on client work, ask:

1. **What's the project type?** — Fixed scope, ongoing retainer, hourly
2. **What's the budget range?** — Affects scope and timeline
3. **Who's the decision maker?** — Direct client vs agency vs stakeholder
4. **What's the timeline?** — Rush job vs flexible
5. **What's the deliverable?** — Code, design, MVP, full product

---

## Dimensions

| Dimension | Spectrum |
|-----------|----------|
| **Engagement** | One-off project ←→ Ongoing retainer |
| **Pricing** | Fixed price ←→ Hourly/value-based |
| **Scope** | Tightly defined ←→ Discovery needed |
| **Client** | Solo founder ←→ Enterprise |
| **Risk** | Low (clear scope) ←→ High (ambiguous) |

---

## Derivation Logic

| If Context Is... | Then Consider... |
|------------------|------------------|
| Unclear scope | Discovery phase first, separate contract |
| Fixed budget, clear scope | Fixed price with change order clause |
| Ongoing relationship | Monthly retainer with scope cap |
| First-time client | 50% deposit, milestone payments |
| Enterprise client | SOW, MSA, longer sales cycle |
| Rush job | 1.5-2x rush rate |

## Part 1: The Proposal

### What to Include

```markdown
# Project Proposal: [Project Name]

## Executive Summary
What you'll build and the outcome for them.

## Scope of Work
- Deliverable 1: Description
- Deliverable 2: Description
- Deliverable 3: Description

## Out of Scope
Explicitly list what's NOT included.

## Timeline
- Week 1-2: Discovery + Design
- Week 3-4: Development
- Week 5: Testing + Launch

## Investment
- Option A: $X (basic)
- Option B: $Y (recommended)
- Option C: $Z (premium)

## Terms
- 50% deposit to start
- 50% on completion
- Revisions: 2 rounds included
- Additional work: $X/hour
```

### Pricing Strategies

| Strategy | Best For | Calculation |
|----------|----------|-------------|
| **Hourly** | Unclear scope, ongoing | Rate × Hours |
| **Fixed** | Clear deliverables | Value / Risk |
| **Value-based** | High-impact projects | ROI percentage |
| **Retainer** | Ongoing relationships | Monthly cap |

### Setting Your Rate

```
Hourly Rate = (Annual Target / Billable Hours) × Buffer

Example:
Target: $150,000/year
Billable hours: 1,200 (not 2,000!)
Buffer: 1.25
Rate: ($150,000 / 1,200) × 1.25 = $156/hour → $150/hour
```

## Part 2: The Contract

### Essential Clauses

1. **Scope** — What you'll deliver (reference SOW)
2. **Timeline** — Start date, milestones, end date
3. **Payment** — Amount, schedule, late fees
4. **Revisions** — How many included, cost of extras
5. **IP Transfer** — When ownership transfers (on payment)
6. **Termination** — Kill fee, notice period
7. **Liability** — Cap at contract value
8. **Confidentiality** — NDA terms

### Payment Terms

| Client Type | Deposit | Structure |
|-------------|---------|-----------|
| **New client** | 50% | 50/50 |
| **Established** | 25-33% | Milestone-based |
| **Enterprise** | Net 30 | Invoice on delivery |
| **High risk** | 100% | Full upfront |

## Part 3: Client Communication

### Weekly Update Template

```markdown
## Week [X] Update

### Completed
- ✅ Task 1
- ✅ Task 2

### In Progress
- 🔄 Task 3 (80%)
- 🔄 Task 4 (50%)

### Blockers
- ⚠️ Waiting on [item] from client

### Next Week
- Task 5
- Task 6

### Budget Status
- Hours used: X of Y
- Remaining: Z hours
```

### Red Flag Management

| Red Flag | Response |
|----------|----------|
| Scope creep | "That's a great idea! Let me add it to the change order." |
| Late payment | Pause work, send reminder, enforce contract |
| Endless revisions | "We've used our 2 rounds. Additional revisions are $X/round." |
| Disappearing client | Document everything, send status emails anyway |
| "Quick favor" | "I'd be happy to! That'll be $X or we can add to next sprint." |

## Part 4: Pricing Psychology

### Anchoring

Always present 3 options:
1. **Basic** — Bare minimum (anchor low)
2. **Standard** — What you actually want them to buy (most value)
3. **Premium** — Aspirational (anchor high)

### Value Framing

Don't say: "It costs $10,000"
Say: "The investment is $10,000, which typically generates $50,000+ in the first year"

### Rush Rates

| Timeline | Multiplier |
|----------|------------|
| Normal (2-4 weeks) | 1x |
| Rush (1 week) | 1.5x |
| Emergency (<3 days) | 2x |

## Part 5: Retainer Model

### Monthly Retainer Structure

```markdown
## Retainer Agreement

Monthly Fee: $X
Hours Included: Y hours
Rollover: No (use it or lose it)
Overages: $Z/hour

Services Included:
- Bug fixes
- Feature updates
- Consultation calls
- Priority response (24h)

Not Included:
- New feature development
- Major redesigns
- Third-party integrations
```

### Retainer vs Project

| Aspect | Retainer | Project |
|--------|----------|---------|
| **Predictability** | High | Variable |
| **Cash flow** | Recurring | Lumpy |
| **Scope** | Flexible | Fixed |
| **Client type** | Ongoing needs | One-time |
| **Risk** | Lower | Higher |
