# Lane Brief Template

Use this when creating a new agent brief.

```md
# <Lane Title>
Status: assigned
Wave: <wave-id>
Owner: <agent/model>
Single source of truth: this file only.

## Explainer
<Plain-language statement of what this lane is responsible for and why it matters.>

## TL;DR
- <Outcome 1>
- <Outcome 2>
- <Outcome 3>

| Field | Value |
|---|---|
| Mission | <full mission> |
| Owned scope | <what this lane owns> |
| Do not touch | <files/areas owned by others> |
| Inputs | <required docs, routes, tables, links> |
| Skills/docs required | <exact skills or reference files> |
| Validation commands | <commands or checks> |
| Done criteria | <acceptance rules> |
| Output contract | Rewrite this same file in place on completion using the completion template. |

## Build Tasks
- [ ] <task 1>
- [ ] <task 2>
- [ ] <task 3>

## Completion Rule
When done, rewrite this file with:
- Explainer
- TL;DR
- required tables
- exact artifact/file paths
- remaining gaps
- task-sheet update row
```
