# Agent Strength Map

Assign lanes to whichever agent you dispatch. The orchestrator does not prescribe specific models — use whatever agent is available and fits the work.

## Assignment Rules

1. Split by product outcome or file boundaries, not by arbitrary subtasks.
2. Give one lane one owner. Never assign the same file to two agents.
3. Reserve one lane for runtime verification or integration review.
4. Reserve the lead orchestrator for review, tracker updates, and redispatch.
