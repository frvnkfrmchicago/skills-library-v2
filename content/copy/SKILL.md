# Copy & Content

Words that convert.

---

## Context Questions

Before writing copy, ask:

1. **What's the product/offer?** — SaaS, physical product, service, info product
2. **Who's the target audience?** — Demographics, pain points, awareness level
3. **What's the desired action?** — Sign up, buy, book a call, download
4. **What's the brand voice?** — Professional, casual, bold, friendly, technical
5. **Where will this appear?** — Landing page, ad, email, social, packaging

---

## Dimensions

| Dimension | Spectrum |
|-----------|----------|
| Tone | Formal/corporate ←→ Casual/conversational |
| Length | Short/punchy ←→ Long-form/detailed |
| Approach | Feature-focused ←→ Benefit-focused |
| Urgency | Soft sell ←→ Hard sell |
| Proof | Claims-based ←→ Evidence-heavy |

---

## Derivation Logic

| If Context Is... | Then Consider... |
|------------------|------------------|
| SaaS + B2B audience | Professional tone, feature-benefit bridge, ROI focus |
| Consumer product + Gen Z | Casual, short, benefit-led, social proof heavy |
| High-ticket service | Long-form, trust-building, objection handling |
| Ad copy + cold traffic | Short, hook-focused, curiosity-driven |
| Email + warm audience | Conversational, personal, direct CTA |

## TL;DR

| Content Type | Purpose |
|--------------|---------|
| **Headlines** | Grab attention, communicate value |

| **CTAs** | Drive action |
| **Features** | Explain what it does |

## Headline Formulas

### Value Proposition

```
[Outcome] + [Without Pain Point]

"Build beautiful apps without writing CSS"
"Get clients without cold calling"
"Ship faster without sacrificing quality"
```

### Transformation

```
[From State] → [To State]

"From idea to production in hours, not weeks"
"Turn sketches into code instantly"
"Transform your workflow with AI"
```

### Quantified Result

```
[Specific Number] + [Outcome]

"10x your development speed"
"Ship 3 features per week"
"Join 50,000+ developers"
```

### Question

```
[Pain Point as Question]

"Tired of writing boilerplate?"
"What if building apps was actually fun?"
"Ready to ship your idea?"
```

### Comparison

```
[Product] vs [Alternative]

"Like Notion, but for developers"
"Figma meets VS Code"
"The anti-template template"
```

## Subheadline Patterns

**Expand the promise:**
```
Headline: "Ship 10x faster"
Subheadline: "AI-powered development that handles the boring stuff 
so you can focus on what matters."
```

**Add specificity:**
```
Headline: "Build beautiful apps"
Subheadline: "Pre-built components, animation presets, and design 
tokens that just work."
```

**Reduce friction:**
```
Headline: "Start building today"
Subheadline: "No credit card required. Free forever for personal projects."
```

## CTA Best Practices

### Action Words

| Weak | Strong |
|------|--------|
| Submit | Get Started |
| Click Here | Try Free |
| Learn More | See How It Works |
| Sign Up | Start Building |
| Buy | Get Access |

### CTA Formulas

```
[Action] + [Benefit]

"Start building for free"
"Get your first client today"
"Deploy in 60 seconds"
```

```
[Action] + [Timeframe]

"Try free for 14 days"
"Ship your first app today"
"Get started in minutes"
```

### Button Hierarchy

| Type | Use | Example |
|------|-----|---------|
| **Primary** | Main action | "Get Started Free" |
| **Secondary** | Alternative | "See Demo" |
| **Tertiary** | Low priority | "Learn More" |

## Feature vs Benefit

**Feature:** What it does
**Benefit:** Why that matters

| Feature | Benefit |
|---------|---------|
| "AI-powered code generation" | "Ship features in hours, not days" |
| "Real-time collaboration" | "Work with your team seamlessly" |
| "One-click deploy" | "Go live instantly, no DevOps needed" |
| "Built-in analytics" | "Know exactly what's working" |

### Writing Benefits

```
So What Test:

Feature: "We use AI to generate code"
So what? "It saves time"
So what? "You ship faster"
So what? "You beat competitors to market"
So what? "You make more money"

Benefit: "Beat competitors to market with AI-powered development"
```

## Section Copy

### Hero Section

```
[Headline]: One sentence, max impact
[Subheadline]: 1-2 sentences expanding the promise
[CTA Primary]: Action + benefit
[CTA Secondary]: Lower commitment option
```

**Example:**
```
Ship 10x Faster with AI

Pre-built components, intelligent code generation, and 
one-click deploy. Build production apps in hours.

[Get Started Free]  [Watch Demo]
```

### Features Section

```
[Section Headline]: Summarize capability set
[Feature Cards]: 
  - Icon
  - Feature name (2-4 words)
  - Description (1-2 sentences)
```

**Example:**
```
Everything you need to ship

[Icon] Smart Components
Drop-in components that adapt to your design system automatically.

[Icon] AI Assistance  
Generate, refactor, and debug code with natural language.

[Icon] One-Click Deploy
Push to production instantly. We handle the infrastructure.
```

### Social Proof Section

```
[Quote]: Specific, results-focused
[Attribution]: Name, role, company (with photo)
[Metric]: Quantified result if possible
```

**Example:**
```
"We shipped our MVP in 3 days instead of 3 months. 
This tool paid for itself on day one."

— Sarah Chen, Founder @ StartupXYZ
```

### Pricing Section

```
[Tier Name]: Clear, memorable
[Price]: Prominent, with billing cycle
[Best For]: Who should choose this
[Features]: What's included
[CTA]: Tier-specific action
```

### FAQ Section

```
[Question]: Real question users have
[Answer]: Direct, helpful, builds trust
```

**Tips:**
- Lead with most common questions
- Address objections (cost, complexity, time)
- Include question about getting help/support

## Tone Guidelines

### Modern SaaS Tone

| Do | Don't |
|----|----|
| Confident | Arrogant |
| Clear | Jargon-heavy |
| Conversational | Stiff |
| Specific | Vague |
| Honest | Hyperbolic |

### Examples by Tone

**Casual/Playful:**
```
"Building apps should be fun. We're making it happen."
"Finally, a tool that doesn't suck."
```

**Professional/Trusted:**
```
"Enterprise-grade infrastructure for modern teams."
"Trusted by 10,000+ companies worldwide."
```

**Bold/Disruptive:**
```
"Traditional development is dead."
"The future of building starts here."
```

## Microcopy

### Form Labels

| Weak | Strong |
|------|--------|
| "Email" | "Work email" |
| "Password" | "Create a password" |
| "Name" | "Full name" |

### Error Messages

| Weak | Strong |
|------|--------|
| "Error" | "That email doesn't look right" |
| "Invalid input" | "Password needs 8+ characters" |
| "Failed" | "Couldn't save. Try again?" |

```

```

```
"Almost there..."
"Loading your dashboard..."
```

### Success States

```
"You're all set!"
"Welcome aboard 🎉"
"Changes saved"
```

## Review Checklist

- [ ] **Clear value prop** - User knows what this does in 5 seconds
- [ ] **Benefits over features** - Focuses on outcomes
- [ ] **Specific, not vague** - Numbers, timeframes, specifics
- [ ] **Action-oriented CTAs** - Tells user what to do
- [ ] **Consistent tone** - Same voice throughout
- [ ] **No jargon** - User understands without context
- [ ] **Scannable** - Works when skimming
- [ ] **Mobile-friendly** - Short enough for small screens
