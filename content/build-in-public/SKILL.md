# Build in Public Skill

Turn your building process into content, community, and credibility.

> **Owner Context:** Frank uses this for all products. Ship updates create accountability and attract early users.

---

## Context Questions

Before building in public, ask:

1. **What are you building?** — App, product, side project, skill set
2. **What's your timeline?** — 1 week sprint, 30-day challenge, ongoing
3. **What platform?** — X/Twitter, LinkedIn, YouTube, personal blog
4. **What's your goal?** — Audience, accountability, early users, job prospects
5. **How much can you share?** — Full transparency, curated updates, results only

---

## Dimensions

| Dimension | Spectrum |
|-----------|----------|
| **Transparency** | High-level updates ←→ Every commit |
| **Frequency** | Weekly ←→ Daily |
| **Depth** | Results only ←→ Full process (failures included) |
| **Platform** | One focused ←→ Cross-posted everywhere |
| **Tone** | Professional/polished ←→ Raw/authentic |

---

## Derivation Logic

| If Context Is... | Then Consider... |
|------------------|------------------|
| Job hunting | Focus on technical depth, portfolio pieces, problem-solving |
| Seeking early users | Focus on pain points, solutions, before/after |
| Building audience | Focus on learnings, tutorials, relatable struggles |
| Accountability only | Focus on daily logs, deadlines met/missed |
| Client work | Be careful what you share, focus on non-NDA wins |

## Ship Thread Template (X/Twitter)

### Daily/Weekly Update

```
Day [X] of building [PROJECT]

✅ What I shipped:
- [Feature/milestone]
- [Feature/milestone]

🔧 What I learned:
- [Insight]

🎯 Tomorrow:
- [Next priority]

[Screenshot/gif if possible]
```

### Launch Thread

```
I just shipped [PROJECT] 🚀

Here's what it does:
[1-2 sentence pitch]

The journey:
- Started: [date]
- Lines of code: [X]
- Biggest challenge: [challenge]
- Stack: [tech stack]

Try it: [link]

Thread with the full story below 🧵
```

### Milestone Thread

```
[PROJECT] just hit [MILESTONE] 🎉

When I started [X weeks/months ago], I had no idea [realization].

Here's what got us here:

1/ [Key decision #1]
2/ [Key decision #2]
3/ [Key decision #3]

Biggest lessons:
- [Lesson 1]
- [Lesson 2]

What's next: [upcoming milestone]
```

## Ship Dates & Accountability

### 30-Day Build Challenge

```
Day 1: Announce publicly what you're building
Day 7: Show first working prototype
Day 14: First user feedback
Day 21: Polish based on feedback
Day 30: Launch publicly
```

### Weekly Rhythm

```
Monday: Set public goals for the week
Friday: Ship thread with what you accomplished
Sunday: Reflect post (optional)
```

## Content Ideas by Stage

| Stage | Content Type |
|-------|--------------|
| Idea | "I'm going to build X because Y" |
| Prototype | "Here's the ugly first version" |
| User Testing | "Real users tried it, here's what happened" |
| Polish | "Before/after of the same feature" |
| Launch | Full launch thread with story |
| Growth | Metrics, learnings, pivots |
| Maintenance | "X months later, what I'd do differently" |
