## The Reasoning Protocol

Every task, every feature, every design decision starts here. Don't skip this.

### 1. Context First

```markdown
□ Who am I building for?
□ What problem am I solving?
□ What does success look like?
□ What are the constraints? (time, budget, tech)
```

### 2. Feeling Check

```markdown
□ What feeling should this evoke?
□ What's the emotional response I want?
□ Is this solving a pain or creating joy?
```

### 3. Innovation Check

```markdown
□ What makes this NOT cookie-cutter?
□ Would someone share this? Screenshot it?
□ Am I following a template or creating something new?
□ Have I considered 3 different approaches?
```


## The Anti-Patterns

Don't do these:

| Anti-Pattern | Why It's Bad |
|--------------|--------------|
| Jump straight to implementation | You'll build the wrong thing |
| Use the first pattern you find | You'll get generic output |
| Skip the feeling check | Your design won't resonate |
| Accept AI's first answer | You're leaving quality on the table |
| Assume one approach is enough | Variety creates innovation |

---

## The Question Loop

Before accepting any output, ask:

```markdown
1. Did this answer my actual question?
2. Is this innovative or predictable?
3. What's missing from this approach?
4. What would make this 10x better?
5. Did I provide enough context?
```

If you answered "no" to any, **iterate**.
